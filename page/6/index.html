Weekly Brackets Extension Update - 14/47

Filed Under Update

Welcome to This Week’s Brackets Extension Updates, Issue 14/47.

Added or updated @Brackets in the week from 11/10/2014 to 11/17/2014.

This week there were 8 new Extensions and 26 updates. This includes themes and other extensions.

What’s going on?

To be honest: I’m sceptical about the download numbers for Brackets File Tree! I don’t know why they have been so high. All the other numbers are in, what I consider a normal range, but Brackets File Tree is somehow way too high.
So, I’m carefully watching the download data for this extension, to figure out if there is any spamming or spoofing going on.

During the next week I’m going to deploy a change to the tweet bot. The change will add a link to a screenshot for a theme (if there is a picture reference in package.json). Let’s see how this works out.
Another change will be a new section with the 11 most downloaded Extension of all time.

So, stay tuned, get the latest Brackets build and let us know what you think.
Thanks!

Filed Under Update

Welcome to This Week’s Brackets Extension Updates, Issue 14/46.

Added or updated @Brackets in the week from 11/03/2014 to 11/10/2014.

This week there were 10 new Extensions and 31 updates. This includes themes and other extensions.

What’s going on?

We made it! We’ve declared Brackets 1.0 last Tuesday in New York during the Future of Web Design Conference. The response was overwhelming and we got a huge spike in Brackets downloads. Brackets 1.0 comes pre-configured with Extract for Brackets (Preview). This extension will help designers and developers alike to access and extract information from a PSD file. Upload a PSD file to the Creative Cloud and try this great feature.

NOTE: You need a free Adobe Creative Cloud Account to use the Extract extension.

The Extract Extension is available from the right toolbar and from the Edit Menu in Brackets. If you have to extract data from PSD, give it a try and let us know what you think of it. We like to hear the good and the bad. Otherwise we can’t improve. ;-)
